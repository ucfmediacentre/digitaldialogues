Wowza Streaming Engine™

Examples

Video On Demand Streaming
Flash HTTP Player | Flash RTMP Player | iOS / Mac OS X | Silverlight | MPEG-DASH Player
This example illustrates how to set up and playback on-demand video. It uses the default stream type.
Live Video Streaming
Flash HTTP Player | Flash RTMP Player | iOS / Mac OS X | Silverlight | MPEG-DASH Player
This example illustrates how to set up and playback live video. It uses the live stream type.
Live nDVR Streaming
Flash HTTP Player | Silverlight Player
This example illustrates how to set up and playback live nDVR video.
SHOUTcast Streaming
Flash RTMP Player
This Adobe Flash example illustrates how to re-stream SHOUTcast and Icecast MP3 or AAC+ audio data through Wowza Streaming Engine. It uses the shoutcast stream type.
Video Chat
Flash RTMP Player
This Adobe Flash example illustrates how to implement video chat between two users. It uses the live-lowlatency stream type and the Camera and Microphone objects to obtain video and audio content. The example can either stream video and audio data between two client connections or loop the data back to itself.
Webcam Recording
Flash RTMP Player | Flash RTMP Player 11
This Adobe Flash example illustrates how to implement a client-to-server webcam recording. It uses the record stream type and the Camera and Microphone objects to obtain video and audio content.
Server Side Modules
Flash RTMP Player
This example is referenced by the Wowza IDE User's Guide and is a good starting point to learn about how to create your first custom server-side module.

Example Installation

  • LINUX
    Run ./installall.sh
  • WINDOWS 7 / WINDOWS 8
    Right-click installall.bat and then select Run as administrator
  • WINDOWS SERVER
    Double-click the installall.bat file
  • MAC OS X
    Double-click the installall.command file

More Info

Note: To run the examples on a machine that does not include a web browser, you must either copy the .html, .js, .gif, .png, .xap, and .swf files to a machine that does include a web browser or make these files available through a web server on the machine running Wowza Streaming Engine. After running the .html file, you will also need to change the rtmp:// URL to point to the IP address of the server running Wowza Streaming Engine.

Note: All Adobe Flash files used in examples are implemented using ActionScript 3.0.
